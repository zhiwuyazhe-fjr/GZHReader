# GZHReader v2.0.0：拥抱纯本地化，告别 Docker 🐳👋 

我们非常激动地宣布 **GZHReader v2.0.0** 正式发布！ 🎉

这是一个里程碑式的大版本。在这个版本中，我们彻底抛弃了以往复杂的环境要求，**移除了对 Docker、WSL 和 MySQL 的依赖**，转而内置了独立的本地 RSS 服务以及轻量级的 SQLite 存储引擎。这不仅让安装配置变得前所未有地简单，也为大家带来了一个更加专注、清爽的“编辑部”工作空间体验。开箱即用，再也不用折腾环境了！

---

## 🌟 核心亮点 (Highlights)

- 🚀 **开箱即用，告别环境折腾**：内置了经过深度整合的 `wewe-rss` 运行时服务。无需下载 Docker，无需拉取镜像，无需配置数据库，真正的“一键启动”。
- ✨ **全新设计的 UI 交互**：告别原有多步向导和繁复的侧边栏控制面板。我们为您带来了安静优雅的“编辑工作区”与独立设置界面，并新增了双主题视觉系统。
- 🗄️ **轻量级数据引擎**：主程序与打包后的 `wewe-rss` 全面转用基于本地 SQLite 驱动的存储方案（双库隔离），系统资源占用断崖式下降。
- 🛠️ **原生 CLI 与服务管理器**：新增本地服务管理器，原生支持服务健康状态检查、PID 管理、日志追踪以及后台面板的智能拉起。

---

## 📝 详细更新日志 (Changelog)

### ✨ 新增 (Added)
- 新增：内置上游服务源码于 `third_party/wewe-rss/`，完全掌控运行环境。
- 新增：全新的本地服务管理器（包含健康状态检查、日志追踪等核心守护功能）。
- 新增：引入 `scripts/build_wewe_rss.ps1` 脚本，用于构建独立的可发行的 `wewe-rss` 运行时。
- 新增：在 PyInstaller 和 Inno Setup 的打包构建链条中完美整合了封装运行时的支持。
- 新增：全新的主页与设置项模板。
- 新增：增加 `THIRD_PARTY_NOTICES.md` 第三方开源协议声明文件，以及设计基准配置记录（`.impeccable.md`）。

### 🔄 优化与更改 (Changed)
- 重构：全面重构了整套后台管理体系，从“Docker 容器接管模式”转向“内置本地服务模式”。
- 调整：使用极简的编辑工作区代替原有的逐步导航安装向导页面（Wizard-style面板）。
- 更改：更新终端 CLI 命令系统，将过往依赖容器的指令规范为本地直读的 `gzhreader service ...`。
- 更改：采用完全隔离挂载的 SQLite 数据库分别承接 GZHReader 与 `wewe-rss` 产生的数据。

### 🗑️ 移除 (Removed)
- 移除：全面拔除对 **Docker Desktop, WSL, MySQL, 和 Compose 脚手架** 的硬性依赖与预设搭建代码。
- 移除：下线由于依赖 Docker 产生的“环境阻断式”落地检测页面。

## 🚀 升级与使用指南

**注意：这是一次包含架构变更的重大更新。**
由于底层存储机制更替为了最直接的独立 SQLite 方案，如果您是从 v1.x 版本迁移过来的老用户，旧的 MySQL 容器数据不会自动继承到本版本中。我们建议将 v2.0.0 视为一次全新的起航。

**快速上手：使用全新的 CLI 命令来管理服务**
```bash
# 启动内置的本地服务
gzhreader service start

# 检查服务运行与健康状态
gzhreader service status
```
