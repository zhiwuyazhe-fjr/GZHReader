按这 5 步完成 Windows 首次安装

很多用户卡在“Docker 下载完了，但还不能启动”。问题通常不是下载按钮，而是 Windows 还没有准备好 WSL2 或虚拟化环境。
完成后回到这里点击“重新检测”
01
下载 Docker Desktop
点击上面的官方下载按钮，先把 Windows 版本安装包跑起来。
02
确认电脑已开启虚拟化
如果提示无法启用虚拟机，通常要去 BIOS / UEFI 打开 Intel VT-x、AMD-V 或 SVM。
03
启用 WSL2 / Virtual Machine Platform
按照 Docker 或微软提示，打开“启用或关闭 Windows 功能”，勾选 WSL 和 Virtual Machine Platform。
04
安装 Linux 发行版或更新 WSL 内核
如果提示缺少 Linux 环境，按微软文档安装一个 Ubuntu 等发行版，或更新 WSL 内核后再启动 Docker。
05
重启电脑并等待 Engine running
首次安装后建议重启一次。重新打开 Docker Desktop，等它显示 Engine running 再回到 GZHReader。

常见卡点

如果你已经下载安装过 Docker，但这个页面仍然提示不可用，通常是下面几种情况。
下载完还不能用
先确认 Docker Desktop 本体已经安装完成，并且桌面程序真正启动了。只下载了安装包但没有继续安装，GZHReader 仍然会检测不到 Docker。
提示缺少 Linux / WSL
这是最常见的首次安装问题。按微软的 WSL 官方文档安装 Linux 发行版，或更新 WSL 内核后再打开 Docker Desktop。
提示需要修改 Windows 高级功能
去“启用或关闭 Windows 功能”里勾选 适用于 Linux 的 Windows 子系统 和 虚拟机平台，然后按提示重启电脑。
启动后一直不 ready
如果 Docker Desktop 长时间卡在 starting，先重启电脑；仍不行时再检查 BIOS 的虚拟化开关是否打开，以及 WSL 是否可正常运行。
